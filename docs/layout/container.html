Wrap any HTML in .ciderui and every native element gets styled automatically — buttons, inputs, tables, typography, and more. The container is centered with responsive max-widths built in.

All element styles use :where() for zero specificity, so any explicit Cider UI class or Tailwind utility always wins. No JS required — pure CSS.

Fixed width

.ciderui is a fixed-width container by default — centered, padded, and responsive. Content snaps to wider max-widths at each breakpoint. 

<div class="ciderui">
  <h1>Hello</h1>
  <p>Centered, responsive, and auto-styled.</p>
  <button>Click me</button>
</div>
Breakpoint Viewport Max width
Default < 768px 576px (36rem)
md ≥ 768px 672px (42rem)
lg ≥ 1024px 896px (56rem)
xl ≥ 1280px 1152px (72rem)

Fluid width

Use .ciderui-fluid for a full-width container with no max-width caps. It provides the same element auto-styling as .ciderui but fills the available space. 

<div class="ciderui-fluid">
  <h1>Full width</h1>
  <p>Stretches to fill the viewport with padding.</p>
</div>

Custom breakpoints

The built-in breakpoints are a sensible default. Override them with Tailwind utilities for full control — utilities always win over component-layer styles. 

<!-- Narrower container -->
<div class="ciderui max-w-lg md:max-w-xl lg:max-w-3xl">
  ...
</div>

<!-- Wider container -->
<div class="ciderui max-w-6xl xl:max-w-7xl">
  ...
</div>

<!-- No max-width at all -->
<div class="ciderui max-w-none">
  ...
</div>

Semantic containers

.ciderui works on any element. Use semantic HTML tags for better accessibility and document structure. 

<body>
  <header class="ciderui py-6">
    <h1>My App</h1>
  </header>

  <main class="ciderui py-8">
    <h2>Welcome</h2>
    <p>All elements inside are styled automatically.</p>
    <form>
      <label for="name">Name</label>
      <input type="text" id="name">
      <button type="submit">Send</button>
    </form>
  </main>

  <footer class="ciderui py-6">
    <small>&copy; 2026 My App</small>
  </footer>
</body>

Styled elements

Every native HTML element inside .ciderui or .ciderui-fluid receives opinionated styling. Here's what each element gets.

Element Styling applied
button Rounded, secondary fill. type="submit" = primary, type="reset" = ghost.
input (text types) Bordered, rounded, card background. Focus ring on :focus.
textarea Same as text input — bordered, rounded, card background.
select Bordered with chevron icon. Dark mode adjusts the icon color.
input[type="checkbox"] Custom checkbox with check icon. Primary fill when checked.
input[type="radio"] Custom radio with thick border when checked.
input[role="switch"] iOS-style toggle switch with sliding knob.
label 13px medium-weight block label.
table Card surface, bordered rows, hover highlight, rounded corners.
hr Thin 1px separator line.
kbd Pill-style keyboard shortcut badge.
progress Thin rounded progress bar with primary fill.
h1 – h4 Apple-style headings with spacing between siblings.
p, a, ul, ol Prose typography — lists get disc/decimal markers, links get underline.
blockquote Left-bordered italic with muted color.
pre, code Rounded with secondary background. pre > code resets to transparent.
fieldset Rounded border with styled legend.
small Muted helper text at smaller size.

Overriding element styles

Because element styles use :where(), they have zero specificity. Any explicit Cider UI class — like btn-primary, btn-outline, or btn-destructive — automatically takes precedence. You never need !important.
