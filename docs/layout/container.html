Wrap any HTML in .ciderui and every native element gets styled automatically — buttons, inputs, tables, typography, and more.

All styles use :where() for zero specificity, so any explicit Cider UI class (like btn-primary) or Tailwind utility always wins. No JS required — it's pure CSS.

Usage

Add the ciderui class to a wrapper element. Everything inside gets styled.

<div class="ciderui">
  <h1>Hello</h1>
  <p>This paragraph is styled automatically.</p>
  <button>Click me</button>
</div>

Styled elements

Every native HTML element inside .ciderui receives opinionated styling. Here's what each element gets.

Element Styling applied
button Rounded, secondary fill. type="submit" = primary, type="reset" = ghost.
input (text types) Bordered, rounded, card background. Focus ring on :focus.
textarea Same as text input — bordered, rounded, card background.
select Bordered with chevron icon. Dark mode adjusts the icon color.
input[type="checkbox"] Custom checkbox with check icon. Primary fill when checked.
input[type="radio"] Custom radio with thick border when checked.
input[role="switch"] iOS-style toggle switch with sliding knob.
label 13px medium-weight block label.
table Card surface, bordered rows, hover highlight, rounded corners.
hr Thin 1px separator line.
kbd Pill-style keyboard shortcut badge.
progress Thin rounded progress bar with primary fill.
h1 – h4 Apple-style headings with spacing between siblings.
p, a, ul, ol Prose typography — lists get disc/decimal markers, links get underline.
blockquote Left-bordered italic with muted color.
pre, code Rounded with secondary background. pre > code resets to transparent.
fieldset Rounded border with styled legend.
small Muted helper text at smaller size.

Overriding with classes

Because .ciderui uses :where(), it has zero specificity. Any explicit Cider UI class — like btn-primary, btn-outline, or btn-destructive — automatically takes precedence. You never need !important.

Tailwind CSS equivalents

.ciderui is a convenience. You can always achieve the same styling with native Tailwind utilities for more control. Here are the key equivalents.

Button

<button class="inline-flex items-center justify-center gap-2
  rounded-xl bg-secondary text-secondary-foreground
  px-4 py-2.5 text-[13px] font-medium
  hover:bg-secondary-hover active:scale-[0.97]
  transition-all">
  Cancel
</button>

Text input

<input class="w-full h-11 rounded-xl border
  border-input-border bg-card px-3.5
  text-[13px] text-foreground outline-none
  hover:border-input-border-hover
  focus:border-primary"
  type="text" placeholder="Jane Appleseed">

Table

<table class="w-full text-left text-[13px]
  rounded-2xl bg-card shadow-sm">
  ...
</table>

Checkbox & Switch

Checkboxes and switches require appearance-none plus SVG background-image tricks for the check icon and knob. These are significantly more complex to replicate with utilities alone — use .ciderui or the dedicated checkbox / switch component classes instead.
